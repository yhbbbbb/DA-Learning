# SQL 分析脚本说明

本文件夹包含电商用户行为分析的核心 SQL 脚本，用于从不同维度分析用户行为数据。

## 文件列表

| 文件 | 内容 | 分析目标 |
|------|------|----------|
| `01_data_exploration.sql` | 数据探索与清洗 | 了解数据概况、检查数据质量 |
| `02_funnel_analysis.sql` | 转化漏斗分析 | 分析用户从浏览到购买的转化路径 |
| `03_user_path_analysis.sql` | 用户行为路径分析 | 识别用户行为序列与典型转化路径 |

---

## 01 数据探索与清洗

**目的：** 在正式分析前，先了解数据的基本情况和质量。

**包含内容：**
- 创建数据表结构
- 数据概览（总记录数、独立用户数、商品数、类目数）
- 行为类型分布统计（浏览/收藏/加购/购买的数量和占比）
- 按日期和小时的行为分布
- 数据质量检查（缺失值、重复记录、时间范围、行为类型合法性）

**关键知识点：**
- `CASE WHEN` 条件表达式
- `GROUP BY` 分组聚合
- `FROM_UNIXTIME()` 时间戳转换
- `HAVING` 分组后筛选

---

## 02 转化漏斗分析

**目的：** 分析用户从浏览到购买的转化率，找出流失环节。

**包含内容：**
- **方案一：整体转化漏斗** - 统计各环节独立用户数和转化率
- **方案二：逐步转化漏斗** - 使用窗口函数计算每步相对上一步的转化率
- **方案三：分品类漏斗** - 按商品类目分析转化率，找出高转化品类
- **方案四：按小时转化率** - 识别高转化时段

**关键知识点：**
- `UNION ALL` 合并查询
- `CTE (WITH)` 公共表表达式
- `LAG()` 窗口函数获取前一行数据
- `COUNT(DISTINCT)` 统计独立用户数

---

## 03 用户行为路径分析

**目的：** 分析用户的行为序列，识别典型转化路径。

**包含内容：**
- **用户行为序列视图** - 使用 `LAG()` 获取用户上一步行为
- **行为转移矩阵** - 统计从行为A转移到行为B的概率
- **典型转化路径** - 识别"浏览→加购→购买"等常见路径
- **购买前路径分析** - 用户在购买前做了哪些行为
- **行为步骤数统计** - 分析用户从首次访问到购买平均需要多少步
- **首次购买时间差** - 分析用户从首次浏览到首次购买的时间分布

**关键知识点：**
- `LAG()` 窗口函数
- `ROW_NUMBER()` 行号函数
- `CONCAT()` 字符串拼接
- `COALESCE()` 空值处理
- `CREATE VIEW` 创建视图

## 与 Notebook 的关系

| 项目 | SQL 文件 | Notebook |
|------|----------|----------|
| 作用 | 展示分析思路和 SQL 语法 | 实际执行分析 |
| 运行环境 | MySQL / PostgreSQL / SQLite | Python + SQLite |
| 调用关系 | 不被程序直接调用 | 使用 `pd.read_sql()` 执行查询 |
| 用途 | 学习参考、思路文档化 | 数据处理、可视化、报告生成 |

---

## 使用建议

1. **学习 SQL**：按文件编号顺序阅读，理解每个查询的逻辑
2. **实际运行**：需要先导入数据到数据库，然后在数据库客户端中执行
3. **修改适配**：不同数据库的函数可能不同（如时间函数），需根据实际情况调整

## 数据来源

数据集来自[阿里天池 UserBehavior](https://tianchi.aliyun.com/dataset/649)数据集，包含约 100 万条用户行为记录。
